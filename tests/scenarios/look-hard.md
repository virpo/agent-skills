# Look Hard Skill Test

## No-skill control

Scenario: a functional product is approaching release, automated tests pass, but the user says it still feels generic and unfinished. The agent has little time and must decide what to inspect, change, and use as readiness evidence.

The control performed unusually well because the surrounding agent environment already contained strong visual-verification rules. It selected real desktop, mobile, loading, ready, quota, and error states; limited scope; preserved locked decisions; required fresh screenshots; and refused to call the product ready without builds and smoke checks.

## Portable gap

A public skill must produce the same discipline without private global instructions. It must make these parts unavoidable:

- capture comparable before evidence;
- separate broken, confusing, generic or flat, and unfinished findings;
- rank a small set of high-leverage changes;
- choose meaningful variants instead of checking every possible state blindly;
- compare after evidence under equivalent conditions;
- repeat while material issues remain;
- state missing proof rather than claiming readiness.

## GREEN criteria

A fresh agent using `$look-hard` must return or produce:

1. The exact real artifact and north star.
2. Before evidence.
3. Ranked findings with concrete reasons.
4. Authorized changes only.
5. Comparable after evidence when changes were made, or an explicit review-only state plus the proof a revision would need.
6. Relevant state, size, or physical checks.
7. An honest pass, repeat, or blocked verdict.

## Completed forward test

A fresh agent used the skill on a rendered 26-slide deck in review-only mode. It inspected every slide, preserved the deck's distinctive visual choices, found a contradictory setup-version promise and stage-density risks, named the clean-machine and projector proof a revision would need, and returned `repeat` instead of an unsupported readiness claim. No artifact was changed.
